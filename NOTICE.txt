Modifications
-------------

The following changes have been made to the upstream live-vlm-webui v0.4.0:

  * Added local-file video/image source support via a new
    LocalFileVideoTrack class in src/live_vlm_webui/local_file_track.py.
    Supports single image files, image folders cycled as slideshows, and
    video files with native-FPS pacing and EOF looping.

  * Extended the WebRTC offer() endpoint in src/live_vlm_webui/server.py
    to dispatch on local_image_path / local_video_path parameters in
    addition to the existing rtsp_url path.

  * Added a /api/browse filesystem endpoint to support an in-browser
    file picker.

  * Added analysis-cadence modes to VideoProcessorTrack in
    src/live_vlm_webui/video_processor.py:
        - "frames"        every N decoded frames (original upstream behavior)
        - "seconds"       every N wall-clock seconds
        - "scene_change"  on detected scene change (mean-pixel-diff
                          on 64x64 grayscale thumbnails, debounced)
    plus Ollama backpressure protection that skips dispatch when the
    previous VLM call is still in flight.

  * Added a Jetson Thor GPU utilization fallback to
    src/live_vlm_webui/gpu_monitor.py that reads cur_freq/max_freq from
    sysfs when jetson-stats does not expose load directly.

  * Added a 4-tab source UI (Webcam / IP Cam / Image / Video) in
    src/live_vlm_webui/static/index.html with an inline file/folder
    browser, cadence-mode controls, and Emplus EEP-500T branding for
    the Jetson Thor system header.

  * Added helper scripts: push.ps1, start.ps1, restart.ps1,
    start (Linux), bin/em_vlm (Linux), bin/install_em_vlm.sh,
    SETUP_THOR.md.
